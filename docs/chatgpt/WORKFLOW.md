# WORKFLOW

## 標準作業フロー

1. `CHATGPT_READ_FIRST.md` を読む。
2. `docs/AI_WORK_STATE.md` を読む。
3. 作業対象に応じて設計文書を読む。
4. 変更範囲を決める。
5. 実装または文書更新を行う。
6. 実行できる検証を実行する。
7. 実行していない検証を未実行チェックへ残す。
8. `docs/AI_WORK_STATE.md` を更新する。
9. 最後に差分ログと次回プロンプトを出す。

## UI作業フロー

1. `docs/DESIGN.md` を読む。
2. 画面目的を確認する。
3. 情報密度、レスポンシブ、アクセシビリティを確認する。
4. コンポーネントを追加または更新する。
5. PC幅とスマホ幅の確認を行う。
6. 未確認の場合は未実行チェックに残す。

## DB作業フロー

1. `docs/DB_SCHEMA.md` を読む。
2. Prisma schemaを確認する。
3. 既存データへの影響を確認する。
4. migrate前に変更意図を明確にする。
5. migrate未実行なら成功扱いしない。
6. seedが冪等か確認する。

## API作業フロー

1. `docs/API.md` を読む。
2. ルーティングとレスポンス形式を確認する。
3. 入力バリデーションを追加する。
4. エラー形式を揃える。
5. DB接続時は認可境界を確認する。
6. 実HTTP smokeを行う。

## コンテンツ作業フロー

1. `docs/CONTENT_POLICY.md` を読む。
2. 公式情報、オリジナル、未確認を分ける。
3. 第三者サイト本文・解説コピーを避ける。
4. `verification_status` を根拠に応じて設定する。
5. 判断できないものは保留にする。

## 課金作業フロー

1. Free / Basicの境界を確認する。
2. ログイン必須と有料必須を分ける。
3. `plans`, `subscriptions`, `entitlements` を確認する。
4. webhook実装時は冪等性を確認する。
5. 決済連携の実通信を行っていない場合は未実行チェックに残す。
